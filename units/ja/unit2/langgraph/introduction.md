# `LangGraph` 入門

<img src="https://huggingface.co/datasets/agents-course/course-images/resolve/main/en/unit2/LangGraph/LangGraph.png" alt="Unit 2.3 サムネイル"/>

次のパートへようこそ。ここでは、複雑な LLM ワークフローの構造化とオーケストレーションを支援するフレームワーク [`LangGraph`](https://github.com/langchain-ai/langgraph) を使って、**アプリケーションを構築する方法**を学びます。

`LangGraph` は、エージェントの流れを**制御**するツールを提供することで、**本番環境対応**のアプリケーションを構築できるフレームワークです。

## モジュール概要

このユニットでは、以下の内容を学びます：

### 1️⃣ [LangGraph とは何か、いつ使うべきか？](./when_to_use_langgraph)
### 2️⃣ [LangGraph の構成要素](./building_blocks)
### 3️⃣ [Alfred：メール仕分け執事](./first_graph)
### 4️⃣ [Alfred：ドキュメント分析エージェント](./document_analysis_agent)
### 5️⃣ [クイズ](./quizz1)

> [!WARNING]
> このセクションの例では、強力な LLM/VLM モデルへのアクセスが必要です。LangGraph との互換性が最も高い GPT-4o API を使用して実行しました。

このユニットを終える頃には、堅牢で整理された本番環境対応のアプリケーションを構築できるようになっているでしょう！

なお、このセクションは LangGraph の入門編であり、より高度なトピックは LangChain Academy の無料コースで学ぶことができます：[Introduction to LangGraph](https://academy.langchain.com/courses/intro-to-langgraph)

それでは始めましょう！
