# LlamaIndex 入門

このモジュールへようこそ。ここでは、[LlamaIndex](https://www.llamaindex.ai/) ツールキットを使って LLM を活用したエージェント（Agent）を構築する方法を学びます。

LlamaIndex は、**インデックスとワークフローを使用して、データ上で動作する LLM 搭載エージェントを作成するための包括的なツールキット**です。このコースでは、LlamaIndex でエージェントを構築するための3つの主要な部分、すなわち **コンポーネント（Components）**、**エージェントとツール（Agents and Tools）**、**ワークフロー（Workflows）** に焦点を当てます。

![LlamaIndex](https://huggingface.co/datasets/agents-course/course-images/resolve/main/en/unit2/llama-index/thumbnail.png)

LlamaIndex の主要な構成要素と、それらがエージェントにどう役立つかを見ていきましょう：

- **コンポーネント（Components）**：LlamaIndex で使用する基本的な構成要素です。プロンプト、モデル、データベースなどが含まれます。コンポーネントは多くの場合、LlamaIndex を他のツールやライブラリと接続する役割を果たします。
- **ツール（Tools）**：検索、計算、外部サービスへのアクセスなど、特定の機能を提供するコンポーネントです。エージェントがタスクを実行するための構成要素となります。
- **エージェント（Agents）**：ツールを使用し、意思決定を行うことができる自律的なコンポーネントです。複雑な目標を達成するためにツールの使用を調整します。
- **ワークフロー（Workflows）**：ロジックを統合して処理するステップバイステップのプロセスです。ワークフローやエージェント的ワークフロー（Agentic Workflows）は、エージェントを明示的に使用せずにエージェント的な振る舞いを構造化する方法です。


## LlamaIndex の特長

LlamaIndex は smolagents などの他のフレームワークと共通する部分もありますが、いくつかの重要な利点があります：

- **明確なワークフローシステム**：ワークフローは、イベント駆動型かつ非同期ファーストの構文を使用して、エージェントがどのように段階的に意思決定すべきかを整理するのに役立ちます。これにより、ロジックを明確に構成・整理できます。
- **LlamaParse による高度なドキュメント解析**：LlamaParse は LlamaIndex 専用に開発されたため、シームレスに統合できます（有料機能です）。
- **豊富なすぐに使えるコンポーネント**：LlamaIndex は長い歴史があり、多くの他のフレームワークと連携できます。そのため、LLM、リトリーバー、インデックスなど、テスト済みで信頼性の高いコンポーネントが多数あります。
- **LlamaHub**：LlamaIndex 内で使用できる数百のコンポーネント、エージェント、ツールのレジストリです。

これらの概念はすべて、有用なエージェントを作成するさまざまなシナリオで必要となります。
以降のセクションでは、これらの概念をそれぞれ詳しく解説していきます。
