# クイズ: AI Agent の評価

このボーナスユニットで扱った Agent のトレーシングと評価の概念について、理解度を確認しましょう。

このクイズは任意であり、採点されません。

### Q1: AI Agent におけるオブザーバビリティ（可観測性）とは主に何を指しますか？
AI Agent のオブザーバビリティの目的を正確に説明している文はどれですか？

- **A)** ログ、メトリクス、スパンを通じて内部の動作を追跡し、Agent の挙動を理解することです。 ✅
  - 💡 正解！オブザーバビリティとは、ログ、メトリクス、スパンを使って Agent の内部動作を明らかにすることを意味します。
- **B)** Agent の実行にかかる金銭的コストの削減のみに焦点を当てています。
  - 💡 オブザーバビリティはコストもカバーしますが、それだけに限定されるものではありません。
- **C)** Agent の外観や UI のみを指します。
  - 💡 オブザーバビリティは UI ではなく、内部プロセスに関するものです。
- **D)** コーディングスタイルやコードの美しさのみに関するものです。
  - 💡 コードスタイルは、この文脈でのオブザーバビリティとは無関係です。


### Q2: Agent のオブザーバビリティで一般的に監視されるメトリクスに含まれないものはどれですか？
オブザーバビリティの対象に通常含まれないメトリクスを選んでください。

- **A)** レイテンシ
  - 💡 レイテンシは Agent の応答性を評価するために一般的に追跡されます。
- **B)** Agent 実行あたりのコスト
  - 💡 コストの監視はオブザーバビリティの重要な側面です。
- **C)** ユーザーフィードバックと評価
  - 💡 ユーザーフィードバックは Agent のパフォーマンス評価にとって重要です。
- **D)** Agent のコード行数 ✅
  - 💡 コード行数は一般的なオブザーバビリティのメトリクスではありません。


### Q3: AI Agent のオフライン評価を最もよく説明しているものはどれですか？
オフライン評価の本質を正しく捉えている文を選んでください。

- **A)** 本番環境で実際のユーザーインタラクションを使って Agent を評価すること。
  - 💡 これはオフライン評価ではなく、オンライン評価の説明です。
- **B)** 既知の正解があるキュレーションされたデータセットを使って Agent のパフォーマンスを評価すること。 ✅
  - 💡 正解！オフライン評価では、テストデータセットを使用して既知の正解に対するパフォーマンスを測定します。
- **C)** Agent の内部ログをリアルタイムで監視すること。
  - 💡 これは評価というよりもオブザーバビリティに関連するものです。
- **D)** 評価メトリクスなしで Agent を実行すること。
  - 💡 このアプローチでは有意義な知見は得られません。


### Q4: Agent のオンライン評価が提供する利点はどれですか？
オンライン評価の利点を最もよく反映している文を選んでください。

- **A)** 事前定義されたデータセットを使用した制御されたテストシナリオを提供します。
  - 💡 制御されたテストはオンライン評価ではなく、オフライン評価の利点です。
- **B)** ライブのユーザーインタラクションと実世界のパフォーマンスデータを取得できます。 ✅
  - 💡 正解！オンライン評価は、本番環境で Agent を監視することで知見を提供します。
- **C)** オフラインテストやベンチマークの必要性を完全に排除します。
  - 💡 オフライン評価とオンライン評価はどちらも重要であり、補完的な関係にあります。
- **D)** Agent の計算コストの削減のみに焦点を当てています。
  - 💡 コスト監視はオブザーバビリティの一部であり、オンライン評価の主な利点ではありません。


### Q5: AI Agent のオブザーバビリティと評価において、OpenTelemetry はどのような役割を果たしますか？
AI Agent の監視における OpenTelemetry の役割を最もよく説明している文はどれですか？

- **A)** コードを計装するための標準化されたフレームワークを提供し、オブザーバビリティのためにトレース、メトリクス、ログの収集を可能にします。 ✅
  - 💡 正解！OpenTelemetry はテレメトリデータの計装を標準化しており、Agent の挙動を監視・診断するために不可欠です。
- **B)** コードの問題を自動的に修正することで、手動デバッグの代替として機能します。
  - 💡 不正解です。OpenTelemetry はテレメトリデータの収集に使用されるものであり、コードの問題をデバッグするためのものではありません。
- **C)** リアルタイム機能を持たず、主に過去のログを保存するデータベースとして機能します。
  - 💡 不正解です。OpenTelemetry はリアルタイムのテレメトリデータ収集と分析ツールへのデータエクスポートに焦点を当てています。
- **D)** モデルパラメータを自動的に調整することで、AI Agent の計算パフォーマンスを最適化するために使用されます。
  - 💡 不正解です。OpenTelemetry はパフォーマンスチューニングではなく、オブザーバビリティを中心としています。
