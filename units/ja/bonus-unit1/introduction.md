

# イントロダクション

![Bonus Unit 1 Thumbnail](https://huggingface.co/datasets/agents-course/course-images/resolve/main/en/bonus-unit1/thumbnail.jpg)

最初の**ボーナスユニット**へようこそ。ここでは、**大規模言語モデル（LLM）を Function Calling 用にファインチューニングする方法**を学びます。

LLM の分野において、Function Calling は急速に*必須の技術*となりつつあります。

そのアイデアは、ユニット1で行ったようなプロンプトベースのアプローチだけに頼るのではなく、Function Calling によってモデルが**トレーニングフェーズ中にアクションを実行し、観測結果を解釈する**ように学習させることで、AIをより堅牢にするというものです。

> **このボーナスユニットはいつ取り組むべきですか？**
>
> このセクションは**オプション**であり、ユニット1よりも高度な内容です。今すぐ取り組んでも、コースを通じて知識が深まってから改めて取り組んでも構いません。
>  
> ただし心配は不要です。このボーナスユニットは必要な情報をすべて含むように設計されているため、ファインチューニングの内部動作をまだ学んでいなくても、Function Calling のためのモデルファインチューニングに関するすべての核心的な概念を一つずつ解説していきます。

このボーナスユニットを理解するための最良の方法は以下の通りです：

1. Transformers を使った LLM のファインチューニング方法を知っていること。まだの場合は[こちらを確認してください](https://huggingface.co/learn/nlp-course/chapter3/1?fw=pt)。

2. `SFTTrainer` を使ったモデルのファインチューニング方法を知っていること。詳しくは[こちらのドキュメント](https://huggingface.co/learn/nlp-course/en/chapter11/1)を確認してください。

---

## 学習内容

1. **Function Calling**  
   現代の LLM が会話を構造化し、効果的に **Tool** をトリガーする方法。

2. **LoRA（Low-Rank Adaptation）**  
   計算コストとストレージのオーバーヘッドを削減する、**軽量で効率的な**ファインチューニング手法。LoRA により、大規模モデルのトレーニングが*より速く、より安く、よりデプロイしやすく*なります。

3. **Function Calling モデルにおける Thought → Act → Observe サイクル**  
   モデルがいつ（そしてどのように）関数を呼び出すか、中間ステップを追跡し、外部の Tool や API からの結果を解釈するかを構造化するための、シンプルかつ強力なアプローチ。

4. **新しい特殊トークン**  
   モデルが以下を区別するための**特殊マーカー**を導入します：
   - 内部的な「思考の連鎖（chain-of-thought）」推論  
   - 外部への関数呼び出し  
   - 外部ツールからの応答

---

このボーナスユニットを終える頃には、以下のことができるようになります：

- Tool に関する API の内部動作を**理解する**。  
- LoRA 技術を使ってモデルを**ファインチューニングする**。  
- Thought → Act → Observe サイクルを**実装**・**修正**して、堅牢でメンテナンスしやすい Function Calling ワークフローを作成する。  
- 特殊トークンを**設計・活用**して、モデルの内部推論と外部アクションをシームレスに分離する。

そして、**Function Calling を行う独自のモデルをファインチューニングできるようになります。** 🔥

それでは、**Function Calling** に飛び込みましょう！
