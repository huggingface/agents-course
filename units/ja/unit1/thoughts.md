

# Thought：内部推論とReActアプローチ

> [!TIP]
> このセクションでは、AIエージェントの内部動作——推論し計画する能力——について深く掘り下げます。エージェントが内部対話を活用して情報を分析し、複雑な問題を扱いやすいステップに分解し、次にどのアクションを取るべきかを決定する仕組みを探ります。
>
> さらに、モデルに行動する前に「ステップバイステップ」で考えることを促すプロンプト技法であるReActアプローチを紹介します。

Thought（思考）は、タスクを解決するための**エージェントの内部推論と計画プロセス**を表します。

これは、エージェントの大規模言語モデル（LLM）が**プロンプトに提示された情報を分析する能力**を活用するものであり、本質的には問題に取り組む際の内なる独白です。

エージェントの思考は、現在の観察を評価し、次に取るべきアクションを決定するのに役立ちます。このプロセスを通じて、エージェントは**複雑な問題をより小さく扱いやすいステップに分解**し、過去の経験を振り返り、新しい情報に基づいて計画を継続的に調整することができます。


## 🧠 一般的な思考タイプの例

| 思考のタイプ       | 例                                                                       |
|--------------------|-------------------------------------------------------------------------|
| 計画               | 「このタスクを3つのステップに分解する必要がある：1) データ収集、2) トレンド分析、3) レポート生成」 |
| 分析               | 「エラーメッセージから判断すると、問題はデータベースの接続パラメータにあるようだ」 |
| 意思決定           | 「ユーザーの予算制約を考慮すると、ミッドティアのオプションを推奨すべきだ」 |
| 問題解決           | 「このコードを最適化するには、まずプロファイリングしてボトルネックを特定すべきだ」 |
| 記憶の統合         | 「ユーザーが以前Pythonを好むと言っていたので、Pythonでの例を提供しよう」 |
| 自己省察           | 「前回のアプローチはうまくいかなかった、別の戦略を試すべきだ」 |
| 目標設定           | 「このタスクを完了するには、まず受け入れ基準を確立する必要がある」 |
| 優先順位付け       | 「新機能を追加する前に、セキュリティの脆弱性に対処すべきだ」 |

> **注意：** Function Calling 用にファインチューニングされたLLMの場合、思考プロセスは省略可能です。詳細はActionsのセクションで説明します。


## 🔗 Chain-of-Thought（CoT）

**Chain-of-Thought（CoT）**は、モデルに**最終的な回答を出す前に、問題をステップバイステップで考えさせる**プロンプト技法です。

通常、次のように始まります：
> *「ステップバイステップで考えてみましょう。」*

このアプローチは、特に論理的・数学的なタスクにおいて、**外部ツールと対話することなく**、モデルが**内部で推論する**のに役立ちます。

### ✅ 例（CoT）
```
Question: What is 15% of 200?
Thought: Let's think step by step. 10% of 200 is 20, and 5% of 200 is 10, so 15% is 30.
Answer: 30
```


## ⚙️ ReAct：推論 + 行動

重要な手法の一つが**ReActアプローチ**で、「Reasoning」（思考）と「Acting」（行動）を組み合わせたものです。

ReActは、モデルにステップバイステップで考えさせ、推論のステップの間にアクション（ツールの使用など）を挟むことを促すプロンプト技法です。

これにより、エージェントは以下を交互に繰り返すことで、複雑なマルチステップタスクを解決できます：
- Thought：内部推論
- Action：ツールの使用
- Observation：ツールの出力を受け取る

### 🔄 例（ReAct）
```
Thought: I need to find the latest weather in Paris.
Action: Search["weather in Paris"]
Observation: It's 18°C and cloudy.
Thought: Now that I know the weather...
Action: Finish["It's 18°C and cloudy in Paris."]
```

## 🔁 比較：ReAct vs. CoT

| 特徴                 | Chain-of-Thought（CoT）     | ReAct                               |
|----------------------|-----------------------------|-------------------------------------|
| ステップバイステップの論理 | ✅ あり                    | ✅ あり                              |
| 外部ツール           | ❌ なし                     | ✅ あり（Actions + Observations）    |
| 適しているタスク     | 論理、数学、内部的なタスク   | 情報検索、動的なマルチステップタスク   |

> [!TIP]
> **Deepseek R1**や**OpenAIのo1**などの最近のモデルは、*回答する前に考える*ようにファインチューニングされています。これらは`<think>`や`</think>`のような構造化トークンを使用して、推論フェーズと最終回答を明示的に分離します。
>
> ReActやCoTがプロンプト戦略であるのに対し、これはモデルが例を通じて思考することを学ぶ**トレーニングレベルの技法**です。
