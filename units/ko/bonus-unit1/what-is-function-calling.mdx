# 함수 호출이란 무엇인가[[what-is-function-calling]]



Function-calling은 **LLM이 환경에서 동작을 수행할 수 있는 방법**입니다. 

처음에는 [GPT-4에서 도입되었으며](https://openai.com/index/function-calling-and-other-api-updates/), 이후 다른 모델에서도 구현되었습니다.
에이전트의 도구처럼, 함수 호출은 모델이 **환경에서 작업을 수행할 능력**을 제공합니다. 

하지만 함수 호출 기능은 **모델에 의해 학습되며**, 다른 에이전트 기법보다 **프롬프트 의존도가 낮습니다**.

Unit 1에서는 에이전트가 **도구 사용을 학습하지 않았고**, 우리는 단지 목록을 제공했으며, 모델이 **이 도구들을 활용해 계획을 일반화할 수 있다는 점에 의존했습니다**. 

여기에서는 **함수 호출을 통해 에이전트를 도구 사용에 맞춰 미세 조정(학습)합니다**。


## 모델은 어떻게 “동작 수행”을 학습할까요?[[how-does-the-model-learn-to-take-an-action]]

Unit 1에서는 에이전트의 일반적인 워크플로우를 살펴보았습니다. 사용자가 에이전트에 도구를 제공하고 질의를 전달하면, 모델은 다음 과정을 반복합니다:

1. *Think*: 목표를 달성하기 위해 어떤 동작을 수행해야 할지 결정합니다.  
2. *Act*: 올바른 인수로 동작을 포맷하고 생성이 중단됩니다.  
3. *Observe*: 실행 결과를 받아옵니다.
