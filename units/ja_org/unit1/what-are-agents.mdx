# エージェント（Agent）とは？

<img src="https://huggingface.co/datasets/agents-course/course-images/resolve/main/en/unit1/whiteboard-no-check.jpg" alt="Unit 1 planning"/>

このセクションを読み終える頃には、エージェントの概念とAIにおけるさまざまな応用について理解できるようになります。

エージェントとは何かを説明するために、まずたとえ話から始めましょう。

## 全体像：エージェントのアルフレッド

アルフレッドを紹介します。アルフレッドは**エージェント（Agent）**です。

<img src="https://huggingface.co/datasets/agents-course/course-images/resolve/main/en/unit1/this-is-alfred.jpg" alt="This is Alfred"/>

アルフレッドが次のような**命令を受け取る**場面を想像してください：「アルフレッド、コーヒーをお願いします。」

<img src="https://huggingface.co/datasets/agents-course/course-images/resolve/main/en/unit1/coffee-please.jpg" alt="I would like a coffee"/>

アルフレッドは**自然言語を理解する**ため、私たちのリクエストをすぐに把握します。

注文を実行する前に、アルフレッドは**推論と計画**を行い、必要なステップとツールを考え出します：

1. キッチンに行く
2. コーヒーマシンを使う
3. コーヒーを淹れる
4. コーヒーを持ち帰る

<img src="https://huggingface.co/datasets/agents-course/course-images/resolve/main/en/unit1/reason-and-plan.jpg" alt="Reason and plan"/>

計画ができたら、**行動に移す必要があります**。計画を実行するために、**自分が知っているツールのリストからツールを使うことができます**。

この場合、コーヒーを作るためにコーヒーマシンを使います。コーヒーマシンを起動してコーヒーを淹れます。

<img src="https://huggingface.co/datasets/agents-course/course-images/resolve/main/en/unit1/make-coffee.jpg" alt="Make coffee"/>

最後に、アルフレッドは淹れたてのコーヒーを私たちのところに持ってきてくれます。

<img src="https://huggingface.co/datasets/agents-course/course-images/resolve/main/en/unit1/bring-coffee.jpg" alt="Bring coffee"/>

これがエージェントです：**推論し、計画し、環境と対話できるAIモデル**のことです。

エージェントと呼ぶのは、_エージェンシー（agency）_、つまり環境と対話する能力を持っているからです。

<img src="https://huggingface.co/datasets/agents-course/course-images/resolve/main/en/unit1/process.jpg" alt="Agent process"/>

## より正式な定義

全体像をつかんだところで、より正確な定義を示しましょう：

> エージェントとは、ユーザーが定義した目的を達成するために、AIモデルを活用して環境と対話するシステムです。タスクを遂行するために、推論、計画、アクションの実行（多くの場合、外部ツールを介して）を組み合わせます。

エージェントには主に2つの部分があると考えてください：

1. **脳（AIモデル）**

すべての思考が行われる部分です。AIモデルは**推論と計画を担当**します。
状況に応じて**どのアクションを取るべきかを決定**します。

2. **身体（能力とツール）**

この部分は**エージェントが実行できるすべてのこと**を表します。

**実行可能なアクションの範囲**は、エージェントが**何を装備しているか**に依存します。たとえば、人間には翼がないため「飛ぶ」という**アクション**は実行できませんが、「歩く」「走る」「跳ぶ」「つかむ」などの**アクション**は実行できます。

### 「エージェンシー」のスペクトラム

この定義に従うと、エージェントはエージェンシーが増加する連続的なスペクトラム上に存在します：

| エージェンシーレベル | 説明 | 呼称 | パターン例 |
| --- | --- | --- | --- |
| ☆☆☆ | エージェントの出力がプログラムフローに影響しない | シンプルプロセッサ | `process_llm_output(llm_response)` |
| ★☆☆ | エージェントの出力が基本的な制御フローを決定する | ルーター | `if llm_decision(): path_a() else: path_b()` |
| ★★☆ | エージェントの出力が関数の実行を決定する | Tool 呼び出し | `run_function(llm_chosen_tool, llm_chosen_args)` |
| ★★★ | エージェントの出力が反復とプログラムの継続を制御する | マルチステップ Agent | `while llm_should_continue(): execute_next_step()` |
| ★★★ | あるエージェントワークフローが別のエージェントワークフローを開始できる | マルチ Agent | `if llm_trigger(): execute_agent()` |

表の出典：[smolagents コンセプトガイド](https://huggingface.co/docs/smolagents/conceptual_guides/intro_agents)


## エージェントにはどのようなAIモデルを使うのか？

エージェントで最もよく使われるAIモデルはLLM（Large Language Model、大規模言語モデル）で、**テキスト**を入力として受け取り、**テキスト**を出力します。

よく知られた例としては、**OpenAI**の**GPT-4**、**Meta**の**Llama**、**Google**の**Gemini**などがあります。これらのモデルは膨大なテキストデータで訓練されており、優れた汎化能力を持っています。LLMについては[次のセクション](what-are-llms)でさらに詳しく学びます。

> [!TIP]
> エージェントのコアモデルとして、テキスト以外の入力も受け付けるモデルを使用することも可能です。たとえば、VLM（Vision Language Model）はLLMのようなモデルですが、画像も入力として理解できます。ここではLLMに焦点を当て、他の選択肢については後ほど説明します。

## AIはどのようにして環境に対してアクションを起こすのか？

LLMは素晴らしいモデルですが、**テキストの生成しかできません**。

しかし、HuggingChatやChatGPTのようなよく知られたチャットアプリケーションに画像の生成を依頼すると、生成できます！どうしてそんなことが可能なのでしょうか？

答えは、HuggingChat、ChatGPTなどのアプリの開発者が、LLMが画像を作成するために使用できる追加機能（**ツール（Tool）**と呼ばれる）を実装したからです。

## エージェントはどのようなタスクを実行できるのか？

エージェントは、**アクション**を完了するために**ツール**を介して実装したあらゆるタスクを実行できます。

たとえば、パソコン上のパーソナルアシスタント（Siriのような）として動作するエージェントを作成し、「マネージャーに今日のミーティングを遅らせるようメールを送って」と頼んだ場合、メールを送信するコードを渡すことができます。これが、エージェントがメールを送信する必要があるときに使用できる新しいツールになります。Pythonで書くことができます：

```python
def send_message_to(recipient, message):
    """受信者にメールメッセージを送信するのに便利"""
    ...
```

LLMは、後ほど説明するように、必要に応じてツールを実行するコードを生成し、目的のタスクを遂行します。

```python
send_message_to("Manager", "Can we postpone today's meeting?")
```

**ツールの設計は非常に重要であり、エージェントの品質に大きな影響を与えます**。タスクによっては非常に特殊なツールを作成する必要がありますが、「web_search」のような汎用ツールで解決できるものもあります。

> **アクションはツールと同じではない**ことに注意してください。たとえば、1つのアクションが完了するために複数のツールを使用することがあります。

エージェントが環境と対話できるようにすることで、**企業や個人が実際に活用できる**ようになります。

### 例1：パーソナルバーチャルアシスタント

Siri、Alexa、Google Assistantのようなバーチャルアシスタントは、ユーザーに代わってデジタル環境と対話する際にエージェントとして機能します。

ユーザーのクエリを受け取り、コンテキストを分析し、データベースから情報を取得し、応答を提供したり、アクション（リマインダーの設定、メッセージの送信、スマートデバイスの制御など）を開始したりします。

### 例2：カスタマーサービスチャットボット

多くの企業が、自然言語で顧客と対話するエージェントとしてチャットボットを導入しています。

これらのエージェントは、質問に答えたり、トラブルシューティングの手順をガイドしたり、社内データベースに課題を登録したり、取引を完了させたりすることができます。

事前に定義された目的には、ユーザー満足度の向上、待ち時間の短縮、売上コンバージョン率の向上などが含まれます。顧客と直接対話し、対話から学習し、時間とともに応答を適応させることで、エージェントの核心的な原則を実践しています。


### 例3：ビデオゲームにおけるAI NPC（ノンプレイヤーキャラクター）

LLMを搭載したAIエージェントは、NPC（ノンプレイヤーキャラクター）をよりダイナミックで予測不能なものにできます。

硬直したビヘイビアツリーに従う代わりに、**文脈に応じて反応し、プレイヤーとの対話に適応し**、より繊細な会話を生成できます。この柔軟性により、プレイヤーの行動に応じて進化する、より生き生きとした魅力的なキャラクターを作り出すことができます。

---

まとめると、エージェントはAIモデル（通常はLLM）をコアの推論エンジンとして使用するシステムで、以下のことができます：

- **自然言語の理解：** 人間の指示を意味のある形で解釈し、応答する。

- **推論と計画：** 情報を分析し、意思決定を行い、問題を解決するための戦略を立てる。

- **環境との対話：** 情報を収集し、アクションを実行し、その結果を観察する。

エージェントとは何かをしっかり理解できたところで、短い確認クイズ（採点なし）で理解を強化しましょう。その後、「エージェントの脳」である[LLM](what-are-llms)について詳しく見ていきます。
