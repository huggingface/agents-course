

# `LangGraph` とは？ [[what-is-langgraph]]

`LangGraph` は [LangChain](https://www.langchain.com/) が開発したフレームワークで、**LLM を統合するアプリケーションの制御フローを管理する**ためのものです。

## `LangGraph` は `LangChain` と違うのか？

LangChain は、モデルやその他のコンポーネントとやり取りするための標準インターフェースを提供し、検索、LLM 呼び出し、Tool 呼び出しなどに役立ちます。
LangChain のクラスは LangGraph 内で使用できますが、必ずしも使用する必要はありません。

パッケージは別々であり、単独で使用することもできますが、実際にはオンライン上のほとんどのリソースが両方のパッケージを組み合わせて使用しています。

## いつ `LangGraph` を使うべきか？
### 制御と自由度のトレードオフ

AI アプリケーションを設計する際、**制御**と**自由度**の間には根本的なトレードオフがあります：

- **自由度**は、LLM がより創造的に振る舞い、予期しない問題に対処する余地を与えます。
- **制御**は、予測可能な動作を保証し、ガードレールを維持することを可能にします。

*smolagents* で見られるような Code Agent は非常に自由度が高いです。単一のアクションステップで複数の Tool を呼び出したり、独自の Tool を作成したりできます。しかし、この動作は JSON で動作する通常の Agent と比べて、予測しにくく制御しにくくなる可能性があります。

`LangGraph` はその対極に位置し、Agent の実行に**「制御」**が必要な場合に真価を発揮します。

LangGraph は**アプリケーションの制御**が必要な場合に特に価値があります。LLM の能力を活用しながらも、予測可能なプロセスに従うアプリケーションを構築するためのツールを提供します。

簡単に言えば、アプリケーションが特定の方法でオーケストレーションされる必要がある一連のステップを含み、各分岐点で意思決定が行われる場合、**LangGraph は必要な構造を提供します**。

例として、ドキュメントに関する質問に回答できる LLM アシスタントを構築したいとしましょう。

LLM はテキストを最もよく理解するため、質問に回答する前に、他の複雑なモダリティ（チャート、テーブル）をテキストに変換する必要があります。ただし、その選択はドキュメントの種類に依存します！

この分岐は決定的（deterministic）ですが、LLM の出力に基づいた条件付き分岐を設計することも可能で、その場合は非決定的（undeterministic）になります。

LangGraph が優れている主なシナリオには以下があります：

- フローを明示的に制御する必要がある**マルチステップの推論プロセス**
- ステップ間で**状態の永続化が必要なアプリケーション**
- **決定的なロジックと AI の能力を組み合わせるシステム**
- **Human-in-the-loop の介入が必要なワークフロー**
- 複数のコンポーネントが連携して動作する**複雑なエージェントアーキテクチャ**

要するに、可能な限り**人間として**、各アクションの出力に基づいてアクションのフローを設計し、それに応じて次に実行するものを決定しましょう。この場合、LangGraph はあなたに最適なフレームワークです。

`LangGraph` は、私の意見では、市場で最もプロダクション対応が進んだエージェントフレームワークです。

## LangGraph はどのように動作するか？

`LangGraph` のコアでは、有向グラフ構造を使用してアプリケーションのフローを定義します：

- **ノード（Node）** は個々の処理ステップを表します（LLM の呼び出し、Tool の使用、意思決定など）。
- **エッジ（Edge）** はステップ間の可能な遷移を定義します。
- **状態（State）** はユーザーが定義し、実行中にノード間で維持・受け渡しされます。次にどのノードに遷移するかを決定する際に、この現在の状態を参照します。

これらの基本的な構成要素については、次の章でさらに詳しく見ていきます！

## 通常の Python と何が違うのか？なぜ LangGraph が必要なのか？

「if-else 文を使った通常の Python コードで、これらのフローをすべて処理できるのでは？」と思うかもしれません。

技術的にはその通りですが、LangGraph は複雑なシステムを構築する際に、素の Python に対して**いくつかの利点**を提供します。LangGraph なしでも同じアプリケーションを構築できますが、LangGraph はより簡単なツールと抽象化を提供してくれます。

状態管理、可視化、ロギング（トレース）、組み込みの Human-in-the-loop などが含まれています。
